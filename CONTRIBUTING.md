# Contributing

Thank you for your interest in contributing to the **Enterprise MikroTik VPN Guide**.

Contributions that improve the quality, accuracy and security of this repository are always welcome.

---

# How You Can Contribute

You can contribute by:

- Reporting documentation errors
- Improving RouterOS reference configurations
- Fixing broken links
- Improving deployment examples
- Suggesting new business scenarios
- Improving security recommendations
- Enhancing troubleshooting documentation
- Correcting technical inaccuracies

---

# Before You Submit

Before opening a Pull Request, please ensure that:

- Documentation is written in clear English.
- RouterOS commands are compatible with the supported RouterOS version.
- Existing formatting and structure are preserved.
- New examples follow the style used throughout the repository.
- All internal links are valid.

---

# Pull Requests

When submitting a Pull Request:

1. Describe the purpose of the change.
2. Keep changes focused on a single topic.
3. Update documentation where necessary.
4. Test RouterOS configurations before submitting.
5. Explain any breaking changes.

Small, well-structured Pull Requests are preferred over large unrelated changes.

---

# Issues

GitHub Issues may be used to report:

- Documentation errors
- Incorrect RouterOS commands
- Broken links
- Typographical errors
- Missing information
- Feature suggestions

Please include as much relevant information as possible.

---

# Coding Style

RouterOS reference configurations should:

- Follow RouterOS 7.x syntax
- Use descriptive comments
- Be modular
- Follow security best practices
- Avoid unnecessary complexity

Documentation should:

- Use Markdown
- Follow the existing heading structure
- Be concise and technically accurate
- Use professional terminology

---

# Code of Conduct

Please be respectful and constructive when participating in discussions.

The goal of this repository is to provide reliable, production-oriented documentation for the MikroTik community.
